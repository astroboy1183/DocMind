DocMind: A Document Intelligence System

DocMind is a retrieval-augmented generation (RAG) system. Its job is to answer
questions about a private collection of documents, and to back up every answer
with citations that point to the exact passages it relied on. Unlike a plain
chatbot, it does not answer from memory — it retrieves real text from your
documents first, then generates an answer grounded only in that text.

The pipeline has four stages. Ingestion parses documents, splits them into
chunks, turns each chunk into a numeric vector with an embedding model, and
stores the result in a searchable index. Retrieval takes a question, finds the
chunks most likely to contain the answer, and reranks them for precision.
Generation assembles those chunks into a prompt and asks a language model to
write a grounded, cited answer. Evaluation measures how good the retrieval and
the answers actually are, so every change can be judged with numbers.

A core principle of DocMind is that quality is dominated by the data, not the
code. If a PDF is parsed badly, or a chunk is split in the middle of a sentence,
no clever prompt will fix the resulting answer. For this reason, the ingestion
stage is treated as the foundation of the whole system, and its output is
inspected by hand early and often.

Another principle is that every component sits behind a small interface. The
embedder, the vector store, the reranker, and the language model can each be
swapped for a different implementation without changing the code that calls
them. This keeps the system flexible: a fast local model can be used while
learning, and a more powerful hosted model can be dropped in later.
